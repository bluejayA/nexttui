## NFR-2 보안 상세: RBAC 3단계 역할 제어 (2026-03-26 확장)

기존 "Keystone 역할 기반 RBAC 메뉴/액션 가시성 제어"를 admin/member/reader 3단계로 구체화.

| 역할 | 메뉴 가시성 | CUD 액션 | Admin 전용 액션 | 최소 권한 원칙 |
|------|-----------|----------|---------------|--------------|
| Admin | 전체 (admin 전용 라우트 포함) | 전부 허용 | 허용 (ForceDelete, Migrate, Evacuate 등) | — |
| Member | 일반 라우트만 (admin 전용 숨김) | Create, Delete 허용 | 차단 → PermissionDenied | — |
| Reader | 일반 라우트만 (admin 전용 숨김) | 전부 차단 → PermissionDenied | 차단 | UI에서 CUD 키 무시 |

**보안 요구사항:**
- 다중 역할 시 최고 권한 적용 (admin > member > reader)
- 알 수 없는 역할은 reader로 처리 (최소 권한 원칙)
- RBAC 판단은 UI 계층(키 무시)과 Worker 계층(PermissionDenied) 이중 가드
- Phase 2 확장: Capability 기반 전환 시 `can_perform()` 인터페이스 유지, 내부 구현만 교체
